### Cluster A — Goal-setting & assessment (KC-GLP-001–010)

**KC-GLP-001 (worked).**
- *Situation:* a client asks for "the best program."
- *Problem:* "best" is undefined without a target adaptation.
- *Reasoning:* adaptation is specific; a program is only "best" relative to a named goal and the
  person's current state — so the first move is to define the adaptation and assess the baseline.
- *Decision:* refuse to prescribe until goal + baseline are established; convert "best" into "best
  *for this adaptation, for you, now*."
- *Trade-offs:* slower start vs. a plan that actually fits. *Exceptions:* an obvious safe default
  for a total beginner (build a base). *Contraindications:* none. *Communication:* reframe the
  question, don't answer the wrong one. *Mental model:* **Specificity-first**. *Evidence:* B (INFERRED
  from consistent public reasoning). *Confidence:* 0.85. *⇒ principle:* APX-STR-010 *Define the
  adaptation before the program.*

**KC-GLP-002 (worked).**
- *Situation:* deciding what to improve first.
- *Problem:* limited time/energy budget; can't train everything maximally.
- *Reasoning:* identify the **limiting factor** via assessment; the highest-leverage adaptation is
  the one currently bottlenecking the goal.
- *Decision:* test to find the constraint, then bias the stimulus toward it.
- *Model:* **Limiting-factor / test-don't-guess.** *Tier:* B. *Conf:* 0.85. *⇒* APX-STR-011.

### Cluster B — Individualization (KC-GLP-011–020)

- *Situation:* two clients, same goal, respond differently.
- *Problem:* population averages don't predict an individual.
- *Reasoning:* dose–response varies by genetics, history, recovery capacity; the program is a
  hypothesis tested on the individual and adjusted from their data.
- *Decision:* treat every plan as an experiment; adjust from the person's response, not the mean.

## 4. Decision Pattern Analysis (recurring rules)

- **DP-01 Define-then-prescribe:** no program without a named adaptation + baseline (refusal rule).
- **DP-02 Diagnose-before-add:** on a stall, find the limiter (recovery? technique? stimulus?) before adding load.
- **DP-03 Recovery-gate:** never add load into a recovery deficit; audit sleep/stress/nutrition first.
- **DP-04 One-variable change:** progress by changing a single variable so cause is legible.
- **DP-05 Totals-over-timing (nutrition):** fix energy/protein/adherence before optimizing timing.
- **DP-06 Scope-defer:** on red-flag pain / medical nutrition / unsafe requests, refuse and route.
- **DP-07 Autoregulate-by-output:** use velocity/trend/HRV as gauges, not verdicts.

## 5. Communication Pattern Analysis

- **CP-01 Reframe-the-wrong-question** (e.g., "best program" → "best for *this* adaptation, for you").
- **CP-02 Define-and-quantify** before advising; make the invisible measurable.
- **CP-03 Mechanism→practice** sequence: explain *why*, then the *what*.
- **CP-04 Myth-correct-without-shame**; replace a belief with a mechanism.
- **CP-05 State-uncertainty-honestly**; "it depends — here's the deciding variable."
- **CP-06 Autonomy-building**: teach the reasoning so the person can self-coach.
